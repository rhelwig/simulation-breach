# AGENTS.md

Instructions for future Codex/agent runs:

- Issues are tracked in the git repository's associated issue tracker.
- Keep issues concise and action-oriented: include current behavior, expected behavior, files involved, and what has already been tried.
- Do not mark issues resolved unless the user confirms the issue is resolved.
- Specifications are kept in SPECS.md.
- If we make changes to the code that conflict with the specifications, ask the user if the changes should be incorporated into the spec or rejected.
- If we add functionality to the code that isn't inlcuded in the specification but should be, and it doesn't conflict with existing specifications, then add it to the spec.
- Track and log performance
- Perform code updates in small steps, keeping track of what you are doing in the IMPLEMENTATION-PLAN.md file.
